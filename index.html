MindsetPerspective
The subtle difference

Developer vs.
Engineer

The Developer

Focuses on mastering specific technologies like React or Node.js. Can build features quickly, but often struggles with underlying system architecture and scalability.

The Engineer

Uses DSA to design robust systems. Thinks about edge cases, performance, and scaling before writing a line of code.

RecognitionCheat Sheet
Read the clues

Keyword
Signals

Every problem has clues hidden in its constraints. Read those before writing a single line of code.

Sort
Sorted array?→ Two Pointers or Binary Search
Sub
Subarray / Substring?→ Sliding Window or Prefix Sum
Freq
Count / Frequency?→ HashMap / HashSet
Path
Shortest path / Connections?→ BFS (unweighted) or Dijkstra (weighted)
Opt
Max / Min with choices?→ Dynamic Programming or Greedy

ExampleOne Pattern, Many Problems
The multiplier effect

Learn one,
solve many

The Two Pointers pattern is one idea. It solves dozens of completely different-looking problems.

P1
Two Sum (Sorted Array)Left pointer at start, right at end. Move based on sum comparison.
P2
Remove Duplicates In-PlaceSlow pointer marks position, fast pointer scans ahead.
P3
Reverse a StringSwap characters from both ends, moving inward until they meet.
P4
Valid PalindromeCompare characters from both ends, skip non-alphanumeric.

ProcessStep by Step
When you see a new problem

How to Pick
a Pattern

01
Understand what is givenIdentify the input shape and expected output clearly before anything else.
02
Read the constraints"n up to 10⁵" rules out O(n²). Constraints tell you which patterns are even possible.
03
Think brute force firstWrite the naive solution mentally. It reveals the redundancy that an efficient pattern removes.
04
Match to a candidate patternSorted array → Two Pointers. Subarray sum → Sliding Window or Prefix Sum. Count → HashMap.
05
Check the pattern's conditionSliding Window requires all-positive values. Two Pointers requires sorted input. Wrong condition = wrong pattern.
